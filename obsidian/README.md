# GanttGen — Obsidian Vault

This vault is the single source of truth for all project documentation, decisions, and development history for GanttGen.

## Navigation

| Folder | Contents |
|---|---|
| `00-project/` | Stable project facts that change infrequently |
| `01-planning/` | Roadmap and prioritized backlog |
| `02-dev-logs/` | Daily session logs — one file per working day |
| `03-decisions/` | Architecture Decision Records (ADRs) |
| `04-references/` | External links, research, and reference material |

## Key files

- `00-project/project-brief.md` — what GanttGen is, its goals, scope, and non-goals
- `00-project/architecture.md` — system design and component breakdown
- `00-project/tech-stack.md` — technology choices and rationale
- `01-planning/roadmap.md` — milestones and timeline
- `01-planning/backlog.md` — prioritized feature and task list

## Conventions

- Dev log files are named `YYYY-MM-DD.md` and live in `02-dev-logs/`.
- ADR files are named `NNN-short-title.md` and live in `03-decisions/`.
- Use `_template.md` files in each folder as the starting point for new notes.
- Cursor reads this vault directly — keep notes accurate and up to date.
