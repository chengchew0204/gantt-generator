# Dev Log — 2026-04-04

## Session goal

Generate a set of example Excel templates that each demonstrate a different project management scenario, for use as sample data and for end-to-end testing.

## Work done

- Created `scripts/generate-templates.mjs` -- a Node.js script that uses the `xlsx` library to produce 8 `.xlsx` files in `excel-template/`.
- Each file contains a "Tasks" sheet (matching the GanttGen column schema) and a "Settings" sheet (24 theme/toggle fields).
- The 8 templates cover:
  1. **Simple Linear Project** -- 7 sequential tasks with finish-to-start dependencies.
  2. **Parallel Workstreams** -- 4 concurrent backend modules converging into integration testing.
  3. **WBS Hierarchy** -- 15 tasks with nested Parent ID structure (1 > 1.1 > 1.1.1).
  4. **Milestones and Baselines** -- zero-duration milestone tasks with baseline start/end dates for variance tracking.
  5. **Critical Path Demo** -- a construction project with parallel MEP trades; demonstrates CPM slack on shorter parallel branches.
  6. **Software Sprint** -- agile-style iteration with mixed categories (Feature, Bug, Tech Debt, Ceremony).
  7. **Event Planning** -- non-software scenario (conference) with Notion Light theme preset.
  8. **Baseline Variance** -- deliberate schedule slippage: actual dates trail baseline dates to demonstrate variance visualization.

## Decisions made

- Used a standalone Node.js ESM script rather than embedding generation in the app, keeping templates as static assets.
- Used `XLSX.write()` with buffer type + `writeFileSync()` instead of `XLSX.writeFile()` to work around SheetJS CE limitation in Node ESM.

## Session 2 — Category Color Palette

### Work done

- Implemented per-category Gantt bar coloring:
  - Added `DEFAULT_CATEGORY_PALETTE` (8 visually distinct colors) and `assignCategoryColors` helper in `App.jsx`.
  - New `categoryColors` state auto-assigns colors from the palette when new categories appear (on import or task edit).
  - `GanttChart` `TaskBar` and `MilestoneDiamond` now use `categoryColors[task.category]` as bar fill, with critical path color still overriding when enabled.
  - `ThemePanel` gains a "Category Colors" section with per-category `<input type="color">` pickers, visible only when categories exist.
  - `ExcelUtils` `SETTINGS_FIELDS` extended with `categoryColors` (JSON string). Import parses JSON; export serializes via `JSON.stringify`.
- Build verified: single-file `dist/index.html` (766 kB) produced with no errors.

### Decisions made

- Category colors are project-specific, not theme-specific: switching theme presets does not reset category colors.
- Palette auto-assignment wraps around the 8-color palette for projects with more than 8 categories.
- Fallback when no category is set: accent color (TaskBar) or warning color (MilestoneDiamond).

## Session 3 — Interactive Guide Tour

### Work done

- Rewrote `src/components/GuideOverlay.jsx` from a static scrollable overlay into an interactive step-by-step tour with spotlight highlighting.
  - 10 tour steps: Welcome, Excel Import/Export, Data Table, Gantt Chart, Dependencies & CPM, Milestones & Baselines, WBS, Theming, View Options, PNG Export.
  - SVG mask-based spotlight: dims the entire screen except the targeted UI element, with an accent-colored pulsing border around it.
  - Tooltip card positioned relative to the highlighted element (bottom/right/left placement depending on the step).
  - Welcome step uses a centered card with no spotlight (no target element).
  - Step navigation: Next/Previous buttons, clickable dot indicators, keyboard support (ArrowRight/ArrowLeft/Escape/Enter).
  - Step counter (e.g. "3 / 10") and progress dots in the footer.
  - Auto-repositions on window resize and scroll.
  - Each step card has an icon, title, description, and bullet-point details list.
- Added `data-guide` attributes to key UI regions for tour targeting:
  - `data-guide="toolbar-import"` on the import/template/save button group in `Dashboard.jsx`.
  - `data-guide="btn-export-png"`, `data-guide="btn-theme"`, `data-guide="btn-view"` on individual toolbar buttons.
  - `data-guide="data-table"` on the DataTable wrapper div in `App.jsx`.
  - `data-guide="gantt-chart"` on the GanttChart wrapper div in `App.jsx`.
- Extended `ActionButton` in `Dashboard.jsx` with optional `guideAttr` prop for passing `data-guide` attributes.
- Existing "Guide" button (BookOpen icon) and `guideOpen` state wiring in `App.jsx` unchanged.

### Decisions made

- Used SVG mask approach for spotlight rather than CSS box-shadow or multiple overlay divs; SVG mask is pixel-perfect and handles rounded corners cleanly.
- Tour steps reference DOM elements via CSS attribute selectors (`[data-guide="..."]`), keeping the guide decoupled from component internals.
- Tooltip placement is computed from the target element's bounding rect at render time and updates on resize/scroll.
- Keyboard navigation (arrows + Escape + Enter) for accessibility.
