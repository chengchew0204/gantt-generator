# Dev Log — 2026-04-01

## Session goal

Session 2 -- Build a fully editable DataTable, upgrade GanttChart to v1 with advanced rendering features, implement WBS parent rows, and wire Dashboard to live computed stats.

## Work done

### App.jsx refactor
- Added `visibleColumns` state (Set) with a sensible default subset (ID, Name, Duration, Start, End, Progress, Status).
- Added `collapsedParents` state (Set) for WBS expand/collapse.
- Implemented `handleUpdateTask(taskId, field, value)` callback -- edits flow through to CPM engine and chart instantly.
- Implemented `handleAddTask()` -- auto-increments ID, appends blank task with "Not Started" status.
- Implemented `handleDeleteTask(taskId)` -- removes task from array.
- Added `buildWbsTree()` -- detects parent tasks by scanning `parentId` references, auto-computes parent start/end/progress from children.
- `displayTasks` filters out children of collapsed parents for both DataTable and GanttChart.
- Defined `ALL_COLUMNS` array with type metadata (text, number, date, status) for the column system.

### DataTable (fully editable)
- Every cell is inline-editable on click: text inputs for strings, number inputs for numerics, native date pickers for dates, `<select>` dropdown for Status.
- Edits commit on Enter or blur, cancel on Escape.
- Parent tasks show auto-computed fields (start, end, progress, duration) as read-only.
- Column visibility picker: gear icon in header opens a checkbox dropdown with all 14 columns.
- Add Row button at the bottom appends a new blank task.
- Delete Row: trash icon appears on row hover.
- WBS expand/collapse: chevron icon on parent rows toggles child visibility.
- Child rows indented with a dash prefix.

### GanttChart v1
- **Milestone diamonds**: Duration=0 tasks render as diamond shapes instead of bars, colored by critical path status.
- **Baseline bars**: thin grey bar rendered below the main task bar when `baselineStart` and `baselineEnd` are present.
- **Dependency arrows**: SVG path lines from predecessor bar end to successor bar start, using an SVG `<marker>` arrowhead. Supports multiple predecessors via comma-separated IDs.
- **WBS summary bars**: parent tasks render as a thin bar spanning the full child date range, with bracket-style end caps.
- **Daily/weekly toggle**: toolbar with Day/Week buttons. Day mode shows individual day columns. Week mode shows week markers with Monday-aligned columns at a narrower unit width.
- **Task labels**: moved to the right of bars (instead of overlapping) for better readability.
- **Date range**: includes baseline dates in min/max calculation to ensure baseline bars are always visible.

### Dashboard enhancements
- All stats are computed reactively via `useMemo`:
  - Overall progress = average of all task progress values.
  - Completed, In Progress, Critical Path counts.
- **Current Task**: first in-progress task sorted by start date.
- **Next Task**: first not-started task sorted by start date.
- Added a vertical divider and two new `TaskInfoCard` components showing current/next task names with truncation.

### Build verification
- `npm run build` produces a single `dist/index.html` at 739 KB (gzip 237 KB).
- Zero lint errors across all edited files.
- Single file output confirmed: `dist/` contains only `index.html`.

## Decisions made

- Column visibility is managed as a `Set` in App state (not persisted to Excel yet -- will be added with the ViewOptions/ThemePanel session).
- WBS parent detection is scan-based: any task whose ID appears in another task's `parentId` field is treated as a parent. No explicit "is parent" flag needed in Excel.
- Dependency arrows use a simple L-shaped path (horizontal-then-vertical-then-horizontal). More sophisticated routing can be added later if arrows cross many rows.
- Task labels are placed to the right of bars on the Gantt chart for readability; they were previously overlaid on the bar itself.
