# Dev Log — 2026-03-30

## Session goal

Project kickoff — define the project, establish the Cursor rules structure, scaffold the Obsidian vault, then scaffold the Vite project and build the core shell.

## Work done

- Created `.cursor/rules/` with four always-on, project-scoped rules:

  - `obsidian-workflow.mdc` — vault layout and reading rules
  - `dev-conventions.mdc` — coding standards and commit conventions

- Populated all project notes with the full project specification.

### Session 1 — Project scaffold and core shell
- Scaffolded the Vite + React project at repo root with Tailwind CSS v4 (`@tailwindcss/vite` plugin).
- Installed all required dependencies: `xlsx` (SheetJS), `lucide-react`, `html-to-image`, `vite-plugin-singlefile`.
- Configured `vite.config.js` with `viteSingleFile()` — verified `npm run build` produces a single `dist/index.html` (722 KB, no external asset files).
- Built `App.jsx` with the full two-section layout:
  - Top: `Dashboard` component with placeholder stats (overall progress, completed count, in-progress count, critical path count) and 4 action buttons (Import Excel, Download Template, Save to Excel, Export PNG).
  - Bottom: horizontal split-pane with draggable resizer — `DataTable` on left, `GanttChart` on right.
- Implemented `ExcelUtils.js` with three functions:
  - `downloadTemplate()` — generates a blank `.xlsx` with all 14 required columns and a hidden Settings sheet stub.
  - `importExcel(file)` — parses `.xlsx`, returns `{ tasks, settings }` from Tasks and Settings sheets.
  - `exportExcel(tasks, settings)` — serializes tasks + settings to `.xlsx` and triggers download.
- Implemented `CpmEngine.js`:
  - Forward pass: earlyStart, earlyFinish for each task.
  - Backward pass: lateStart, lateFinish from project finish date.
  - Total float = lateStart - earlyStart; tasks with float === 0 marked `isCritical: true`.
  - Topological sort via Kahn's algorithm with cycle fallback.
  - Supports comma-separated predecessor IDs in the `dependency` field.
- Applied the Linear/Notion/Tremor dark aesthetic:
  - CSS custom properties for all theme colors on `:root`.
  - Custom scrollbar styling.
  - Tabular-nums for all numeric columns.
  - Status badges with semantic colors, progress bars, critical path indicators.
- `DataTable` renders task rows with ID, name, duration, dates, progress bar, status badge, and critical path flag.
- `GanttChart` renders SVG timeline with day headers, month labels, weekend shading, task bars (with progress fill), critical path coloring, and a "today" dashed line.
- Confirmed the built `dist/index.html` contains zero external CDN/network calls — all URLs inside are from bundled library internals (SheetJS namespace URIs, Tailwind license comment).

## Decisions made

- Cursor rules are placed in `.cursor/rules/` inside the project root so they are project-scoped and do not affect other workspaces.
- Obsidian vault lives at `obsidian/` so Cursor can read and write it directly.
- Excel file (with a hidden "Settings" sheet) is the sole persistence mechanism — no localStorage, no server.
- Used Tailwind CSS v4 with the `@tailwindcss/vite` plugin (no `tailwind.config.js` needed).
- Used Node.js 22 via `fnm` (no system-level install required).
- SVG vs Canvas decision still open for the Gantt renderer; current implementation uses SVG.

## Blockers & open issues

- SVG vs Canvas decision needed for performance with large task counts (see open question in `architecture.md`).
- Zoom/pan approach not yet decided.
- `DataTable` is read-only; inline editing will be added in a future session.
- Theme presets and custom color builder not yet implemented.
