# Dev Log — 2026-04-05

## Session goal

Translate a construction project Gantt chart (Chiller & CW305 installation) from a reference image into a GanttGen Excel template. Add click-to-fill-date feature on the Gantt chart grid.

## Work done

- Added template `09-chiller-cw305-construction.xlsx` to `scripts/generate-templates.mjs`.
- The template covers 33 tasks across 9 WBS work packages: Civil Works, Structural Steel & Gantry, POD Renovation, Equipment Installation, Chilled Water Piping, Electrical Power, Fire Protection, Inspection/Testing, and Commissioning.
- 11-week schedule from 2026-03-09 to 2026-05-22 with full WBS parent/child hierarchy and inter-task dependencies.
- Ran the generator; output file: `excel-template/09-chiller-cw305-construction.xlsx`.
- Added click-to-fill-date feature on the Gantt chart grid:
  - When the user clicks to edit a date cell (Start Date or End Date) in the DataTable, the Gantt chart enters date-picking mode.
  - Clicking on any date column in the timeline header or body grid fills the active date field for the selected task.
  - Visual feedback: crosshair cursor and accent-colored column highlight with floating label (e.g., "Start: 2026-04-07") only appears during active date picking, not when a task is merely selected.
  - Task bar / milestone clicks stop propagation to avoid triggering date-fill on existing bars.
- Refactored date-pick activation: introduced `datePickField` state in `App.jsx`, threaded through `DataTable` and `GanttChart` as a prop. The DataTable sets it when a date cell enters edit mode; the GanttChart reads it to know which field to fill and clears it after committing.
- Fixed click-to-fill-date not working due to two race conditions:
  1. Date input `onBlur` was clearing `datePickField` before the Gantt grid click could fire. Fixed by not clearing pick mode on blur (only on explicit Enter commit or Escape cancel).
  2. Task bars and milestones had `onClick stopPropagation` which blocked grid clicks when picking. Fixed by rendering a transparent overlay rect on top of the body SVG during pick mode that catches all clicks.
- Added drag-to-reorder for task rows in the DataTable. Users can grab the grip handle on the left gutter of any non-parent row and drag it to a new position. A 2px accent-colored drop indicator shows the target position. No new dependencies added (native mouse events only).
- Fixed deselection: clicking empty space in the Gantt grid, timeline header, DataTable body, or below task rows now deselects the selected task.
- Changed default view options: Critical Path and Slack Bars are hidden by default.

## Decisions made

- Used WBS numbering (1.0, 2.1, etc.) matching the source chart rather than sequential integers, preserving the original project structure.
- Mapped the weekly bar positions from the image to calendar dates starting on Mondays with weekends skipped.
- Click-to-fill-date reuses the existing `onUpdateTask` callback which already handles auto-calculation of duration and end-date via `DateUtils`, keeping the feature consistent with manual editing in the DataTable.
- Date-pick mode is explicitly driven by `datePickField` state rather than inferred from task selection, giving precise control over when the Gantt grid is clickable.
